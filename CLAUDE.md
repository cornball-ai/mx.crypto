# mx.encrypt — local conventions

R wrapper around the Rust `vodozemac` crate (Matrix Olm + Megolm) via
dbdahl's cargo-framework / roxido. Crypto only — no HTTP, no JSON
canonicalisation. Sibling to `mx.api`.

## Layout

- `R/` — R wrappers + roxygen. `R/roxido.R` is auto-generated by
  `build.rs` from `roxido.txt`; do not edit. Hand-written wrappers in
  `account.R`, `olm.R`, `megolm.R`.
- `src/rust/` — Rust crate. `src/rust/roxido/`, `src/rust/roxido_macro/`,
  `src/rust/rbindings/` are vendored from cargo-framework; do not edit.
  `src/rust/src/lib.rs` is ours.
- `src/r_init_shim.c` — calls `R_init_mx_encrypt_rust()` (note:
  underscore, not dot — R substitutes `_` for `.` in init symbols).
- `tools/configure.R`, `configure`, `cleanup` — copied from salso.
- `src/rust/vendor.tar.xz` — `cargo vendor` output, built locally,
  shipped in tarball, never in `vendor/` form in git.

## Build

```bash
# Local dev
R CMD INSTALL .

# Tinyverse equivalents (preferred)
r -e 'tinypkgr::install()'
r -e 'tinytest::test_package("mx.encrypt")'

# Re-vendor after Cargo.toml changes
cd src/rust && cargo vendor && tar -cJf vendor.tar.xz vendor && rm -rf vendor
```

## Editing R wrappers

Each `#[roxido]` Rust fn is callable from R as `.Call(.fnname, args)`
once the package is built. The auto-generated `R/roxido.R` defines the
`.fnname` symbols (via `useDynLib(mx.encrypt, .registration = TRUE)`).
Hand-written wrappers in `R/account.R` etc. call those.

## CRAN gotchas inherited from salso's pattern

- `vendor.tar.xz` ships in tarball; `vendor/` is extracted by configure
  and cleaned up after. Build is hermetic, no network.
- `CARGO_HOME` is redirected to `src/rust/.cargo-home` so we don't
  write to `~/.cargo`.
- `.cargo/config.toml` (inside vendor.tar.xz) caps cargo jobs at 2
  per CRAN policy.
- MSRV declared in `SystemRequirements: rustc (>= 1.85)` and matches
  vodozemac's `rust-version`.

## Out of scope for 0.1.0

- `m.room_key_request` / `m.forwarded_room_key`
- Cross-signing (master/self/user-signing keys)
- SAS verification (`m.key.verification.*`)
- libolm-format pickle import (modern serde pickles only)
